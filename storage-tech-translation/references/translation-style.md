# Translation Style

Use natural Chinese for Minecraft storage-technology readers.

- Prefer mechanism-aware translation over literal translation.
- Keep the original structure of technical documents: headings, numbered steps, lists, code blocks, item counts, and links.
- Translate normal prose smoothly; avoid English word order.
- Use official Minecraft Chinese names only for official game content.
- Use storage-tech community wording for mechanisms and device roles.
- Keep design names, unstable abbreviations, schematic names, and project names in English.
- For first appearances of important terms, use either `中文（English）` or `English（中文解释）` depending on which is more natural.
- For repeated terms, use the established Chinese wording or the preserved English term consistently.
- For Discord/chat logs, remove reaction UI, "add reaction", "forward", "more", duplicated quote blocks, and irrelevant server badges.
- When a term is uncertain, preserve English and add a brief note rather than inventing a fixed translation.

Common output patterns:

- Direct translation: translation first, optional notes after.
- Explanation request: explain the mechanism first, then translate key phrases.
- Terminology request: provide recommended Chinese, whether to keep English, and a short storage-tech explanation.
